Example 2

  • A relational schema R =(A, B, C)
  • Functional dependencies: AB → C, C → B
  • Keys ?
  • If AB → C is valid in R and it covers entire relational schema then its left hand side is a minimal key (A, B)
  • If C → B then through augmentation rule AC → AB
  • If AC → AB is valid in R and it covers entire relational schema then its left hand side is a minimal key (A, C)
  • Normal form ?
  • Not BCNF because left hand side of C → B is not a minimal key
  • 3NF because right hand side of C → B is a prime attribute
  • Decomposition into BCNF ?
  • R1 = (C, B), R2 = (A, B)

Example 3

  • A relational schema R = (A, B, C)
  • Functional dependencies: AB → C, C → B, C → A
  • Keys ?
  • If AB → C is valid in R and it covers entire relational schema then its left hand side is a minimal key (A, B)
  • If C → B and C → A then through union rule C → AB
  • If C → AB is valid in R and it covers entire relational schema then its left hand side is a minimal key (C)
  • Normal form ?
  • BCNF because left hand side of each functional dependency is a minimal key

Example 4

  • A relational schema R = (A, B, C)
  • Functional dependencies: A → B
  • Keys ?
  • If A → B is valid in R then through augmentation rule AC → BC
  • If AC → BC is valid in R and it covers entire relational schema then its left hand side is a minimal key (A, C)
  • Normal form ?
  • not 2NF because a nonprime attribute B functionally depends (A → B) on a subset of primary key (A, C)
  • Decomposition into BCNF ?
  • R1 = (A, B), R2 = (A, C) or
  • R1 = (A, B), R2 = (B, C)

Example 6

  • A relational schema R = (A, B, C)
  • Functional dependencies: A → B, B → C
  • Keys ?
  • If A → B and B → C then through transitivity rule A → C
  • If A → B and A → C then through union rule A → BC
  • If A → BC is valid in R and it covers entire relational schema then its left hand side is a minimal key (A)
  • Normal form ?
  • Not 3NF because a non prime attribute C is transitively dependent on primary key A
  • 2NF because no nonprime attribute depends on a part of primary key

Example 7

  • A relational schema R =(A, B, C, D)
  • Functional dependencies: A → B, A → C, B → D
  • Keys ?
  • If A →B and A → C then through union rule A → BC
  • If A → B and B → D then through transitivity rule A → D

  • Not 3NF because a non prime attribute D is transitively dependent on primary key A
  • 2NF because no nonprime attribute depends on a part of primary key

Example 8

  • A relational schema R =(A, B, C, D)
  • Functional dependencies: A → B, B → D, C → B
  • Keys ?
  • If A → B and B → D then through transitivity rule A → D
  • If A → D and A → B then through union rule A → BD
  • If A → BD then through augmentation rule AC → BCD
  • If AC → BCD is valid in R and it covers entire relational schema then its left hand side is a minimal key (A, C)
  • If C → B and B → D then through transitivity rule C → D

  • If C → BD then through augmentation rule AC → ABD
