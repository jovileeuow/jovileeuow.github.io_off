This presentation shows how JSON notation is used as a formal specification of a logical data model based on a concept of nested/hierachical type of data structures.


JSON is an acronym formed from the initial letters of Java Script Object Notation (JSON).


JSON is is a light–weight data interchange format.


JSON is based on a subset of the JavaScript Programming Language Standard ECMA-262 3rd Edition - December 1999. It is why JSON is a perfect solution for elimination of "impedance mismatch" problem. Impedance mismatch occurs when a notation use for representation of a logical schema and DDL/DML statements on the instances of logical schema are incompatible with a notation of a programming language that hosts DDL/DML statements. For example SQL statement versus a hot programming language like Java or Python or C++.


JSON data model is built on two data structures:

(1)a collection of name:value pairs called as an object,

2)an ordered list of values called an array.


In JSON notation an object is an unordered set of name:value pairs and an array is a sequence of values. A value is either an atomic value like number, string or boolean value or it can be an object or array itself. In this way the values associated with the names can be nested in JSON notation to any possible level.


Below are the simple examples of flat objects, i.e. name:value pairs where a value is atomic.

{"first name": "James"} -> an object represent information about the first name of a person.

{"city":"Sydney", "street":"Victoria", "building number":25} -> An object represents an address that consists of city name, street name and building number.

{"number":"25", "isOdd ?":true, "nothing":null} An object represent a variety of values: a number (floating point number), a Boolean value, and no value at all.

{ } -> An object is empty, it contains no name:value pairs.

In JSON a value associated with a name can be either a string or a number or true or false or null or an object or an array.


Below we provide few examples of "exotic" values being string of control characters.

{"single quote": "\'", "back slash":"\\", "forward slash":"\/", "line feed":"\u000A" }


The first value associated with a name "single quote" is a string that consists of a single quotation. A back slash (\) escapes interpretation of a single quotation as a string delimiter.


The second value associated with a name " back slash" is a string that consists of a single back slash character. Like before the first back slash escapes interpretation of the second back slash as a character that escapes from a functional role played by a character.


The third value associated with a name "forward slash" is a string that consists of a single forward slash character. Like before the first back slash escapes a functional interpretation of the second forward slash.


The fourth value associated with a name "line feed" is a string that consists of a single line feed character expressed is Unicode system of characters.


The next object

{"fraction":0.25, "true":true, "false":false, "nothing": null}

consists of four name:value pairs. The fist pair associates a floating point number with a name "fraction". The second and third pairs associate Boolean values with the names "true" and "false". The fourth pair associates nothing represented by a null symbol with a name "nothing".


The next object

{"full name": {"first name":"James",

"initials":null,

"last name":"Bond"},

"number":"007"}

is an example of a nested object where a value associated with a name "full name" is an object itself. A nested object consists of three name:value pairs that represent the first name, initials and last name of a person. An outer object consists of two name:value pairs. A name "full name" is associated with an object and a name "number" is associated with a string "007".


The next object

{"colours":["red", "green", "blue"]}

consists of one name: value pair where a name "colours" as associated with an array of strings. In this case a value associated with a name is not an atomic value and instead it is an array of atomic values.


The last object


{"numbers":[10, 20, 30, 40, 50]}


also consists of one name:value pair. A name "number" is associated with an array of integer numbers [10, 20, 30, 40 50]. In this case a value associated with a name is an array.

BSON is an abbreviation of Binary JSON.


BSON is a computer data interchange format used mainly as a data storage and network transfer format in the MongoDB database system.


BSON is a is a binary encoded serialization of JSON-like documents (objects). In BSON JSON objects are called as documents and name:value pairs are called as key:value pairs.


The main difference between BSON and JSON is inclusion of the extensions not in JSON that allow for representation of more data types. For example BSON has a Date type and a BinData type


The common data types for BSON and JSON include: null, boolean, number, string, array, embedded document(object)


The new data types include: date, regular expression, object id, binary data and code.


BSON ? What is it ?

  • Binary JSON (BSON) is a computer data interchange format used mainly as a data storage and network transfer format in the MongoDB database system
  • BSON is a is a binary encoded serialization of JSON-like documents (objects)
  • BSON includes the extensions not in JSON that allow for representation of data types
  • For example BSON has a Date type and a BinData type
  • The common data types include: null, boolean, number, string, array, embedded document(object)
  • The new data types include: date, regular expression, object id, binary data, code

A key "_id" play a very important role in each BSON document.


First, every document (object) in BSON must have an "_id" key and "_id" key cannot be associated with no value (null symbol) !

If a new document does not have information about a value associated with "_id" key then a database system implementing a data model of BSON documents automatically generates a value unique within a collection and assigns such value to '_id" key.


A value associated with "_id" key's can be of any type

A method ObjectId() is used to provide a default value associated with "_id" key when such value is not provide in BSON document.

In a collection of documents (objects) every document (object) must have a unique value for "_id", If a value is generated automatically then a database system must generate a unique value. If a value is provided within a document then a database system must verify the uniqueness of the value. To spped this process up the values associated with a ket "_id" are indexed.


If a document has no "_id" key then when it is inserted into a collection "_id" key is automatically added to the document


In the following BSON document a value associated with "_id" key is automatically generated by a method ObjectId().

{"_id":ObjectId()}


In the following BSON document a value associated with "_id" is generated by a method ObjectId() from a string ""507f191e810c19729de860ea".

{"_id":ObjectId("507f191e810c19729de860ea") }


In the following BSON document a value associated with "_id" is an integer number 1234567.

{"_id": NumberInt(1234567) }


In the following BSON document a value associated with "_id" is a string "1234567 CSCI235 01-AUG-2021". A string is created through concatenation of student number 1234567, subject code CSCI235 and enrolment date 01-AUG-2021.

{"_id": "1234567 CSCI235 01-AUG-2021"}
